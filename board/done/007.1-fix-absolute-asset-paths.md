# 007.1 — Fix absolute asset paths for GitHub Pages

Sub-ticket of epic `007-fix-github-pages-white-page`. Relative `./assets/` URLs break on nested routes (e.g. `/experience`) because the browser resolves them against the current path segment. Root-absolute `/assets/` paths resolve correctly from any route.

## Acceptance criteria

- [x] Layout profile image and beach CSS variable use `/assets/...` (not `./assets/...`)
- [x] About Me, Dogs, and Coding Reference pages and tests use `/assets/...` for images
- [x] `src/data/jobs.ts` logo paths use `/assets/...`
- [x] `Layout.test.tsx` expects root-absolute profile src and beach backdrop CSS variable
- [x] Experience-entry skill docs reference `/assets/` and `public/assets/` for logo files

## Resolution notes

Replaced `./assets/` with `/assets/` across layout, pages, tests, and jobs data after resetting to `origin/main` (post stash-conflict cleanup). Added beach backdrop unit test in `Layout.test.tsx`.
